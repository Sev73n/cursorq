## 目录结构（zip 解压后）

```
cursorq/
  CursorQ.exe          # 主程序
  content/             # 内置默认内容（离线可用）
    manifest.json
    copy/              # 文案
    mascot/            # 占位图 + 动图
  data/
    app-state.json     # 用量、语言等（自动生成）
    content-sync.json  # 内容版本（自动生成）
  config/
    remote.json        # 可选在线更新（默认关闭）
  logs/
    cursorq.log
  tools/
    sqlite3.exe        # 内置 SQLite 工具（支持大体积数据库）
  scripts/
    refresh-usage.mjs
  node_modules/
  README.txt           # 本说明文件
```

## 使用前

1. 本机需 **Node.js 20+**（读取 Cursor 用量）与已登录 **Cursor 桌面版**。
2. 双击 `CursorQ.exe` 即可运行；启动即用 `content/` 预设，**无需联网**。
3. 可选：`config/remote.json` 开启后，约 **30 秒** 联网 **合并** 远程新 joke/gif（不覆盖本地与手动添加）。

## 快速开始

1. 解压 `cursorq-0.2.1-win64.zip` 到任意目录
2. 确保已安装 Node.js 20+ 并登录 Cursor 桌面版
3. 双击 `CursorQ.exe` 启动
4. 胶囊会显示在屏幕顶部，显示用量信息

## 功能说明

### 胶囊显示

| 颜色 | 含义 |
|------|------|
| 绿 → 蓝 | 周期剩余 + 节余银行占订阅额度的比例 |
| 橙 | 总量超前日均 + 今日用量 ≥ 2× 日预算，但周期余量充足（温和提醒） |
| 红 | 总量超前日均 + 今日用量 ≥ 2× 日预算，且周期余量紧张（紧急警告） |

总量未超前日均时，无论今日多用多少都保持蓝/绿色。

### 详情面板

- **日预算 / 今日用量 / 周期用量 / 剩余天数**
- 按 API / Auto 分组，可展开查看分模型明细
- 底部提示行连点三下可进入调试模式（模拟用量状态）

### 操作方式

| 操作 | 效果 |
|------|------|
| 拖动胶囊 | 移动窗口位置（按住约 0.5 秒后开始） |
| 单击吉祥物 / 双击胶囊 | 展开/收起详情面板 |
| 单击文案 | 切换下一条段子/状态文案 |
| 双击吉祥物 | 切换下一张动图 |

### 系统托盘

左键单击打开菜单；双击显示胶囊。菜单含：显示/隐藏、中/英文、总是置顶、开机启动、立即刷新、同步文案/动图、退出。

## 远程合并（可选）

- 只 **追加** 远程新内容；已有本地文件、用户自增 gif **不会被覆盖**。
- 托盘 **「同步文案/动图」** 可随时手动触发合并。
- 未开启或离线时始终使用安装包内的 `content/`。

## 数据安全

- 仅读取 Cursor 本地数据库，**不修改**任何数据
- 仅向 Cursor 官方服务器请求用量信息
- 所有数据仅保存在本机，不向第三方传输
